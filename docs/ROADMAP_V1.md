# AmarktAI Network V1 Roadmap

The phases describe sequencing, not permission to create parallel systems.
Every phase extends the same Brain, registries, router, jobs, and artifacts.

## Phase 0: Operating Truth

- Operating Truth
- Documentation
- AI Context

Exit: future work can identify canonical rules without chat history.

## Phase 1: Provider Truth Layer

- Provider Registry
- Capability Registry
- Model Discovery
- Routing Profiles

Exit: providers and capabilities have one truthful runtime source, and model
selection is discovery-driven.

## Phase 2: Brain Completion

- Research Engine
- Artifacts
- Memory
- Adult
- Agents

Exit: these systems use the canonical Brain and return truthful results, jobs,
artifacts, or blockers.

## Phase 3: Provider Completion

- Hugging Face
- Together
- Groq
- GenX
- Qwen
- MiMo

Exit: provider-native contracts are implemented and proven without presenting
unverified capabilities as working.

## Phase 4: Dashboard

- Command Center
- Studio
- Providers
- Models
- Artifacts
- Research
- Agents
- Apps
- Brands

Exit: the dashboard displays backend truth without adding a second execution or
registry layer.

## Phase 5: Website

- Landing
- Capabilities
- Providers
- Architecture
- Roadmap

Exit: public claims are derived from proven product truth.

## Phase 6: Marketing App Integration

- Brands
- Campaigns
- Research
- Publishing
- Analytics

Exit: the Marketing App calls scoped Brain capabilities and never providers.
